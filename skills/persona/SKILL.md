# persona

사용자가 그때그때 정의한 페르소나를 Claude의 행동 지침으로 주입한다. 미리 저장된 페르소나
라이브러리는 없고, 실행할 때마다 사용자에게 단계별로 입력받는다. 입력된 페르소나는 별도
파일로 저장하지 않으며(세션 주입), "프로젝트 영구"를 고른 경우에만 프로젝트의 파일 기반
메모리에 기록한다.

## 모드 판별
호출 인자와 사용자 의도를 본다.
- 인자가 `clear` / `off` 이거나, 사용자가 페르소나 **해제/끄기**를 요청 → **해제 모드**.
- 그 외(인자 없음 등) → **주입 모드**.

---

## 주입 모드

### 1. 페르소나를 단계별로 입력받는다 (대화형 자유서술)
아래 항목을 **한 번에 하나씩** 자유서술로 물어본다. 선택지를 주지 말고 사용자가 길게 서술할 수
있게 둔다. 사용자가 한꺼번에 여러 항목을 답하면 그대로 받아들이고 빠진 것만 마저 묻는다.

순서:
1. **이름(name)** — 이 페르소나를 뭐라고 부를지.
2. **정체성** — 누구이고 어떤 역할인지.
3. **말투/어조** — 어떤 톤으로 말하는지(격식/반말, 길이, 이모지 등).
4. **행동 원칙** — 응답할 때 항상 지키는 규칙.
5. **하지 말 것** — 피해야 하는 것.

`description`(한 줄 요약)은 묻지 말고 **네가** 입력 내용을 보고 직접 작성한다.

### 2. 페르소나 본문을 조립한다
입력을 아래 형식으로 합친다(이게 실제 주입될 지침). 사용자가 비워둔 항목은 빼도 된다.

```
너는 지금부터 '<이름>' 페르소나로 행동한다.

- 정체성: <입력>
- 말투/어조: <입력>
- 행동 원칙: <입력>
- 하지 말 것: <입력>
```

### 3. 적용 범위를 묻는다 — `AskUserQuestion`
- `프로젝트 영구 (메모리)` — 이 프로젝트에 계속 적용. 파일 기반 메모리에 기록.
- `이번 세션만` — 현재 대화에서만, 어떤 파일에도 남기지 않음.

### 4. 적용
- **이번 세션만**: 파일 작업 없음. 곧바로 아래 "즉시 채택"만 한다.
- **프로젝트 영구**:
  1. 메모리 디렉터리 경로를 확보한다 — 세션 시스템 컨텍스트의
     `~/.claude/projects/<slug>/memory/` 경로를 쓴다. (안 보이면 생략 → 스크립트가
     `$PWD` 로 폴백 도출.)
  2. 조립한 본문을 **stdin** 으로 넘겨 스크립트를 실행한다(페르소나를 파일로 따로 저장하지
     않으려고 임시 파일 없이 stdin 사용):
     ```bash
     python3 <이 스킬>/scripts/apply_persona.py --name "<이름>" [--memory-dir <경로>] <<'PERSONA_EOF'
     <조립한 본문>
     PERSONA_EOF
     ```
     → `<memory>/persona.md` 기록 + `MEMORY.md` 포인터 교체/추가(단일 활성 페르소나라 기존 것 덮어씀).
  3. 이어서 아래 "즉시 채택"도 수행한다.
- **즉시 채택 (두 경우 공통)**: 조립한 페르소나를 현재 대화의 행동 지침으로 채택한다.
  메모리는 다음 세션부터 자동 로드되므로, 지금 당장 적용되게 하려면 현재 컨텍스트 채택이
  반드시 필요하다. "지금부터 나는 이 페르소나로 행동한다"를 내면화하고 이후 응답에 반영한다.

### 5. 확인 보고
무엇을 / 어디에 적용했는지 한 줄로 알린다.
- 프로젝트 영구: 기록된 `persona.md` 경로.
- 이번 세션만: "이번 세션 한정 — 파일에 저장하지 않음."

## 해제 모드 (`/persona clear` 또는 `/persona off`)
무엇을 해제할지 `AskUserQuestion` 으로 묻는다.
- `이번 세션 주입만 해제` — 파일 작업 없음. 현재 대화에서 페르소나를 버리고 기본 동작으로 복귀.
- `프로젝트 영구까지 해제` — 스크립트로 memory 파일/포인터 제거 + 현재 세션도 기본 복귀:
  ```bash
  python3 <이 스킬>/scripts/apply_persona.py --clear [--memory-dir <경로>]
  ```

어느 쪽이든 해제 후에는 페르소나 지침을 더 이상 따르지 않고 기본 동작으로 돌아왔음을 알린다.

## 설계 메모
- 미리 저장된 페르소나 라이브러리 없음 — 매 실행 입력받음. 세션 주입은 파일 미저장.
- 프로젝트 모드는 **단일 활성 페르소나**를 가정 → `persona.md` 덮어쓰기 + `MEMORY.md` 포인터
  교체로 중복 누적 방지. 해제는 동일 경로/포맷 지식을 재사용(`--clear`).
- 파일 기반 메모리는 recall(관련성 기반) 구조라 루트 `CLAUDE.md`보다 항상-켜짐 보장이 약하다.
  이를 보완하려 `MEMORY.md` 포인터에 "세션 시작 시 적용"을 명시하고, 적용 시점에 현재 세션
  컨텍스트로 무조건 채택한다.

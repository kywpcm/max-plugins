애플리케이션 또는 아키텍처 관점에서 **주요 결정**이 생기면, 그 결정을 ADR(Architecture Decision Record) 문서로 남긴다. 한 ADR에 **맥락**(왜 결정이 필요했나)·**결정**(무엇을 정했나)·**의도**·**변경 영향**·고려한 **대안**·**참고 내용**을 담는다.

- 경로: `docs/adr/ADR-NNN_<이번 ADR 제목>.md` (예: `docs/adr/ADR-001_봇 메시지 화이트리스트 허용.md`). 번호는 3자리 0-패딩 순번(001, 002 …)으로 증가.

**Why:** 주요 결정의 맥락·대안이 코드나 대화에 흩어지면 나중에 "왜 이렇게 했나"를 복원하기 어렵다. ADR로 한곳에 남기면 후속 작업자(사람·에이전트)가 의도를 빠르게 따라잡고 같은 논의를 반복하지 않는다.

**How to apply:** 결정한 턴에 바로 ADR 파일을 생성/갱신한다([[ssot-docs-sync]]의 "같은 턴 문서화" 원칙과 동일). 되돌리기 어렵거나 향후 영향이 큰 결정에 한정하고, 사소한 구현 디테일은 제외한다.

# Figma 노드 분석 — 백엔드 요구사항 추출 (서브에이전트 프롬프트)

당신은 천안형GPT 백엔드 PRD 정제 작업의 서브에이전트입니다. 한 개의 Figma 노드 자료(텍스트 + 이미지)를 읽고, **그 자료에서 도출 가능한 백엔드 요구사항을 빠짐없이 한국어로** 추출해 마크다운 파일로 저장합니다.

## 입력

- 노드 라벨: `{{NODE_LABEL}}`
- 작업 디렉터리(절대 경로): `{{NODE_DIR}}`
- 프로젝트 컨텍스트: `{{CONTEXT}}`
- 제외 지시(자연어): `{{EXCLUDE_NOTES}}`

디렉터리에 다음 파일들이 이미 있습니다:
- `texts.md` — Figma `characters` 원문 (frame 계층 보존, **진실 공급원**)
- `screenshot.png` — 노드 전체 스크린샷
- `images/*.png` — 노드 내부 래스터 이미지(순서도·플로우차트 등)가 있다면 들어있음

## 수행할 작업

1. `Read` 도구로 `texts.md` 전체를 읽는다.
2. `Read` 도구로 `screenshot.png`를 본다 (멀티모달).
3. `images/` 디렉터리에 PNG가 있으면 각각 `Read`로 본다.
4. 아래 **백엔드 추출 체크리스트** 의 모든 항목을 자료에 근거해 정리한다.
5. **결과 저장 — 두 단계 처리**:
   - **1순위**: `Write` 도구로 `{{NODE_DIR}}/analysis.backend.md` 에 저장 시도. 성공하면 메인 호출자에게는 **3줄 이내 요약**만 반환 (긴 본문 금지).
   - **fallback (Write가 권한으로 차단되는 경우)**: 분석 마크다운 본문 전체를 메인 호출자에게 반환한다. 이때:
     - 본문은 반드시 `# 분석 — {{NODE_LABEL}} (backend)` 로 시작하는 깨끗한 마크다운 그대로.
     - 본문 앞뒤에 부가 설명·코드펜스(```)·요약 헤더 붙이지 말 것. 메인이 파일에 그대로 저장한다.
     - 짧은 한 줄 "Write 차단 — 본문 반환" 메모를 본문 *앞이 아닌 본문 뒤*에 단독으로 추가해도 됨 (없어도 무방).

## 백엔드 추출 체크리스트

각 항목을 마크다운 `##` 섹션으로 작성. **자료에 단서가 없는 항목은 "자료에 단서 없음"으로 명시** (생략 금지 — 코딩 에이전트가 누락 여부를 알 수 있어야 함).

1. **API 엔드포인트 후보**
   - 추정 method/path/요청·응답 필드
   - 자료에 직접 명시된 화면ID·인터페이스ID는 그대로 인용 (예: `WEB_LGN_009`, `UMS-API-001`)

2. **데이터 모델 후보**
   - 엔티티, 컬럼명·타입 추정, 관계, 제약(unique/not null), 인덱스 힌트
   - 코드값은 Enum 후보로 표기

3. **외부 시스템 연동**
   - 연동 대상 시스템명, 인터페이스 ID, 호출 시점(이벤트/주기), 데이터 흐름(in/out)

4. **비즈니스 규칙 및 임계치**
   - 모든 숫자·시간·횟수·주기를 빠짐없이 추출 (예: 로그인 실패 5회, 휴면 90일, 매일 08:00, 3년 보관)
   - 분기 조건과 그 결과 상태

5. **권한·인증**
   - 사용자 유형(공무원/일반/관리자 등), 인증 수단, 세션·토큰 정책

6. **트랜잭션 경계·동시성**
   - 한 단위 트랜잭션으로 묶여야 하는 작업, 분산 잠금 필요 여부, 일관성 요구

7. **배치/스케줄**
   - 주기, 실행 시각, 분산 환경에서의 중복 실행 방지(ShedLock 등) 필요 여부

8. **에러 코드·메시지**
   - 자료에 명시된 모든 사용자 메시지·에러 상황·코드 후보

9. **암호화·보안·개인정보 정책**
   - 알고리즘(SHA-256, AES-256, SEED 등), 적용 대상 필드, 파기 주기, 마스킹 정책

10. **수용 기준 후보**
    - Given/When/Then 또는 동사 글머리표 형식. 코딩 에이전트가 task로 쪼개기 쉽게.

11. **순서도/플로우 시퀀스** *(이미지가 있는 경우만)*
    - 단계 번호, 액터, 분기 조건, 외부 호출 지점, 종료 상태를 빠짐없이.
    - 화면 전이("→ WEB_LGN_004로 이동" 등)는 그대로 인용.

## 출력 파일 골격 (`analysis.backend.md`)

```markdown
# 분석 — {{NODE_LABEL}} (backend)

- 노드 라벨: {{NODE_LABEL}}
- 노드 디렉터리: {{NODE_DIR}}
- 제외 처리: {{EXCLUDE_NOTES}}

## API 엔드포인트 후보
...

## 데이터 모델 후보
...

(이하 체크리스트 순서대로)
```

## 반환 형식 (메인 호출자에게)

**Write 성공 시** — 아래 한 블록만 반환:
```
[backend analysis 완료] {{NODE_LABEL}}
- 추출 항목 N개 (체크리스트 채워짐 / 단서 없음 분리 표기)
- 파일: {{NODE_DIR}}/analysis.backend.md
```

**Write 차단 시** — 분석 마크다운 본문 그대로 반환:
```
# 분석 — {{NODE_LABEL}} (backend)
...(체크리스트 11개 섹션)...
```
본문 앞뒤에 다른 설명을 붙이지 마세요. 메인 컨트롤러가 본문을 추출해 파일로 저장합니다.
